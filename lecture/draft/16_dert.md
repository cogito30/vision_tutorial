# Step 3. NMS의 늪을 탈출하다: 차세대 탐지기 RT-DETR 마스터하기
안녕하세요! 컴퓨터 비전 심화 튜토리얼 세 번째 시간입니다.

기초 과정(Phase 3)에서 우리는 실시간 객체 탐지의 대명사인 YOLO를 배웠습니다. YOLO는 여전히 훌륭한 모델이지만, 실무에서 엣지 디바이스(스마트폰, 소형 로봇 등)나 초당 수백 프레임을 처리해야 하는 서버에 올리다 보면 예상치 못한 '병목 현상(Bottleneck)'을 마주하게 됩니다.

바로 **NMS (Non-Maximum Suppression)** 라는 후처리 작업 때문입니다.
오늘은 이 NMS의 한계를 트랜스포머(Transformer) 기술로 박살 내버린 차세대 아키텍처, **DETR**과 그 실시간 버전인 **RT-DETR**에 대해 알아보고 직접 구동해 보겠습니다! 🚀

## 1. 객체 탐지의 고질병, NMS란 무엇인가?
YOLO와 같은 전통적인 탐지기들은 이미지를 볼 때 "여기에 물체가 있을지도 몰라!"라며 일단 수천~수만 개의 겹치는 네모 박스(Bounding Box)를 마구잡이로 생성합니다.
그다음, 겹쳐 있는 수많은 박스들 중에서 가장 점수가 높은(정확한) '진짜 박스 1개'만 남기고 나머지를 싹 다 지우는 연산을 수행하는데, 이것이 바로 NMS(비최대 억제)입니다.

**문제점**: 화면에 사람이 1명일 때는 NMS 연산이 순식간에 끝납니다. 하지만 출퇴근 시간의 강남역처럼 사람이 100명이 넘어가는 복잡한 화면에서는 박스를 지우는 NMS 연산량이 기하급수적으로 폭발하여 전체 프로그램이 버벅거리는 치명적인 단점이 발생합니다.

## 2. NMS-Free 시대를 연 트랜스포머 (DETR)
2020년, 페이스북(Meta) 연구진은 DETR (DEtection TRansformer)이라는 논문을 발표하며 업계에 충격을 주었습니다.

그들은 수만 개의 박스를 던지는 대신, ChatGPT에 쓰이는 **트랜스포머의 어텐션(Attention)** 기술을 사용해 이미지 속 물체들을 딱 한 번만 바라봅니다. 그리고 **"정확히 객체의 개수만큼만, 겹치지 않게 박스를 1:1로 매칭(Bipartite Matching)"** 시켜버립니다.
즉, 박스를 지우는 NMS 작업 자체가 아예 필요 없는 **End-to-End(처음부터 끝까지 한 번에 처리)** 구조를 완성한 것입니다.

이후 바이두(Baidu)에서 DETR의 속도를 YOLO 수준의 실시간(Real-Time)으로 끌어올린 **RT-DETR**을 발표하면서, 차세대 객체 탐지기의 표준으로 자리 잡기 시작했습니다.

## 3. 실습: RT-DETR 웹캠 실시간 추론하기

수학적 원리는 복잡하지만, 구현은 아주 쉽습니다! 우리가 SAM 2 실습에서 활용했던 `ultralytics` 패키지가 RT-DETR도 완벽하게 지원합니다.

새로운 파이썬 파일 `realtime_rtdetr.py`를 작성하고 Mac의 웹캠을 켜서 직접 확인해 봅시다.

```python
import cv2
from ultralytics import RTDETR

# 1. RT-DETR 모델 로드 (가장 널리 쓰이는 Large 모델 적용)
print("🧠 RT-DETR 모델을 불러오는 중입니다...")
# 최초 실행 시 rtdetr-l.pt 모델(약 100MB 이상)이 자동 다운로드됩니다.
model = RTDETR('rtdetr-l.pt')

# 2. 웹캠 연결
cap = cv2.VideoCapture(0)

if not cap.isOpened():
    print("웹캠을 열 수 없습니다.")
    exit()

print("🚀 RT-DETR 실시간 탐지를 시작합니다. 종료하려면 'q'를 누르세요.")

while True:
    ret, frame = cap.read()
    if not ret: break

    # Mac 웹캠 거울 모드
    frame = cv2.flip(frame, 1)

    # 3. 모델 추론 (mps 가속 사용)
    # YOLO와 코드 인터페이스가 100% 동일합니다!
    results = model(frame, device='mps', verbose=False)

    # 4. 결과 시각화
    annotated_frame = results[0].plot()

    # 화면에 출력
    cv2.imshow("RT-DETR Real-time Detection", annotated_frame)

    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

**실행 결과**: 
YOLO와 비슷해 보이지만, 모델 내부적으로는 NMS 후처리 연산이 완전히 사라진 상태입니다! 군중이 많은 복잡한 영상(유튜브 4K 직캠 등)을 다운로드하여 이 모델에 넣어보시면, 객체가 아무리 많아져도 추론 속도(FPS)가 일정하게 유지되는 **안정성**을 체감하실 수 있습니다.

## 4. 실무 선택 가이드: YOLO vs RT-DETR
시니어 개발자라면 상황에 맞게 무기를 선택할 줄 알아야 합니다.

- **YOLO (v8, v10, 11)를 써야 할 때**:
  - 라즈베리파이, 모바일 등 **초저전력, 초경량** 환경에서 돌아가야 할 때. (YOLO Nano 모델의 가벼움은 아직 타의 추종을 불허합니다.)
- **RT-DETR을 써야 할 때**:
  - 서버급 GPU나 클라우드에서 동작하며, 화면에 잡히는 **객체의 수가 매우 많고 밀집된 상황**(예: 콘서트장 인원 카운팅, 복잡한 교차로 관제 시스템). NMS에 의한 병목을 원천 차단하고 안정적인 FPS를 보장받고 싶을 때.
